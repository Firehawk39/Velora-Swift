# Contributing to Velora

First off, thank you for considering contributing to Velora! It's people like you that make Velora such a great tool for everyone.

## How Can I Contribute?

### Reporting Bugs
*   Check the [Issues](https://github.com/yourusername/velora/issues) to see if the bug has already been reported.
*   If not, open a new issue. Include a clear title, a description of the bug, and steps to reproduce it.

### Suggesting Enhancements
*   Open an issue to discuss your idea before implementing it.
*   Clearly explain the value of the enhancement and how it fits the [Vision](THE_VISION).

### Pull Requests
1.  Fork the repo and create your branch from `main`.
2.  If you've added code that should be tested, add tests.
3.  Ensure the test suite passes.
4.  Make sure your code lints.
5.  Issue that pull request!

## Code Style
*   **Web:** We use Prettier and ESLint. Please run `npm run lint` before submitting.
*   **Swift:** Follow standard Swift API Design Guidelines. Keep UI components reusable.

## Our Vision
Always keep the [The Vision](THE_VISION) in mind. Velora is about **ownership, privacy, and cinematic experiences**.
